---
title: Configuration
description: Configure your gittydocs site.
---

Create `gittydocs.jsonc` in your docs folder:

```jsonc
// docs/gittydocs.jsonc
{
  "site": {
    "name": "Site Name",
    "repo": {
      "owner": "username",
      "name": "repo",
      "ref": "main",
      "docsPath": "docs"
    }
  },
  "nav": [
    {
      "label": "Section",
      "items": [
        { "label": "Page", "path": "/" },
        { "label": "Subpage", "path": "/subpage" }
      ]
    }
  ],
  "links": {
    "issues": "https://github.com/user/repo/issues"
  }
}
```

## Options

| Field | Type | Description |
|-------|------|-------------|
| `site.name` | string | Site title |
| `site.repo.owner` | string | GitHub username or org |
| `site.repo.name` | string | Repository name |
| `site.repo.ref` | string | Branch or tag (default: `main`) |
| `site.repo.docsPath` | string | Path to docs folder |
| `nav` | array | Custom navigation sections |
| `links.issues` | string | URL to issues page |

## Writing Content

Use `.md` or `.mdx` files. Frontmatter optional:

```mdx
---
title: Page Title
description: Description for SEO
---

# Heading

Content here.
```

Files sorted by:
1. `index.*` first
2. Numeric prefix (`01-intro.md`)
3. Alphabetical

## Customize

The template is a thin wrapper around [Velite](https://velite.js.org/). For full control:

```bash
bunx degit blankeos/gittydocs/apps/web my-docs
cd my-docs && bun install
```

Edit `velite.config.ts`, components, styles. Deploy the same way.
